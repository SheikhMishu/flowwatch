POST-LAUNCH IMPROVEMENTS (April 2026)
=======================================

- Monthly AI usage limits implemented (Pro: 100/mo, Team: 500/mo)
  ai_usage table tracks per-org monthly Anthropic calls.
  Cached responses never count against quota.
  DB failure during usage check blocks the request rather than bypassing the limit.

- Logging gaps fixed across 4 files:
  incident-grouping.ts  — console.error replaced with logger.error + full context.
  alert-engine.ts       — per-alert try/catch so one failure never blocks others.
  ai-debug.ts           — getCached, storeResult, getMonthlyUsage all log DB errors.
  email.ts              — SES errors logged with recipient + subject before re-throwing.

- Super Admin Portal built at /admin:
  /admin           — Overview: orgs, users, MRR, AI calls, alerts, instances, signups
  /admin/orgs      — All orgs: plan, members, instances, AI usage, Stripe status
  /admin/orgs/[id] — Full org detail: members, instances, 9-metric stats, AI chart
  /admin/users     — All users with org memberships and role badges
  /admin/usage     — 4 Recharts charts: alert firings, executions, signups, AI by month
  /admin/visitors  — Full visitor analytics (see below)
  API routes under /api/admin/* — all check is_super_admin from DB (not JWT)

- Visitor tracking built:
  Middleware fires non-blocking fetch to /api/track on every page visit.
  /api/track parses UA (browser/OS/device/bot detection), geo-looks up IP via
  ip-api.com (free, 2s timeout), stores to page_visits table.
  /admin/visitors shows: stats (total/today/week/unique IPs), 30-day trend chart,
  top pages, countries with flags, device breakdown, browsers, referrers,
  searchable visit log with full details (IP, location, device, browser, OS, referrer).
