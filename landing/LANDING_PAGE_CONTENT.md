**Headline:** n8n is powerful — but when something breaks, **you're blind.**

**4 problem cards:**

1. **No central visibility**
   Workflows everywhere. No single source of truth.

2. **Failures get buried in logs**
   n8n logs errors. It doesn't surface patterns, group failures, or flag what keeps recurring.

3. **Debugging takes too long**
   Find the execution. Open the node. Repeat. Fully manual. Completely slow.

4. **Clients expect answers instantly**
   When it breaks, someone's waiting. Without visibility, you're always last to know.

## SECTION: Features

**Headline:** Everything you need. Nothing you don't.

**5 feature cards (in this order):**

### 1. Incident Detection *(badge: Your edge)*
Stop chasing logs. FlowMonix groups repeated failures by error pattern into a single incident — with frequency, timeline, and auto-resolution built in.
- Groups failures by error signature
- Shows frequency + timeline in one view
- Auto-resolves after 30 min of stability

### 2. One-Click Retry
Identify the issue, fix it, re-run the workflow — without opening n8n. Resolve incidents end-to-end from a single dashboard.
- Retry directly from incident cards
- No need to context-switch to n8n
- Available to viewers — read + retry only

### 3. AI Debugging *(badge: Pro)*
Know what broke and exactly how to fix it — without digging through node outputs or reading cryptic stack traces.
- Root cause in plain English
- Step-by-step fix suggestions
- Cached per error — no redundant API calls

### 4. Smart Alerts
Get notified only when it actually matters. Real pattern-based triggers with built-in cooldowns so your team never gets spammed.
- Trigger on real failure patterns
- Slack, email, and webhook channels
- Per-alert cooldowns — no spam

### 5. Public Status Page *(badge: Agencies)*
Give clients confidence without extra work. Auto-generated status pages show health, incidents, and uptime — no login required.
- Custom slug per workspace
- Live incident count + instance health
- Auto-refreshes every 60 seconds
